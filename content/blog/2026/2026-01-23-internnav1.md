mini分支中：
- VLN-N1: 100.7GB（这个是训练System1和System2的数据）
- VLN-CE: 23.6GB（Habitat的验证数据）
- VLN-PE: 94.8GB（InternUtopia即Isaac的验证数据）
由于我们只有200GB（想不花钱在租赁服务器里存储的最大空间），而mini分支大约218GB，所以我们不可能下载整个mini分支。因此，我们必须选择性下载。

```bash
# 1. 清理现有的Git配置
git config --global --unset credential.https://hf-mirror.com.username
git config --global --unset credential.helper

# 2. 创建一个包含Token的URL
# 将 <YOUR_TOKEN> 替换为你的HuggingFace Token
YOUR_TOKEN="hf_your_actual_token_here"  # 替换为实际token

# 3. 使用带Token的URL进行克隆
GIT_LFS_SKIP_SMUDGE=1 git clone -b v0.5-mini "https://{username}:${YOUR_TOKEN}@hf-mirror.com/datasets/InternRobotics/InternData-N1"
```

使用 GIT_LFS_SKIP_SMUDGE=1 克隆，这只下载了文件索引。接下来可以通过`git lfs pull --include="preview/**" --include="vln_ce/traj_data/**"`继续拉取，但是这太BYD慢了，故而尝试 hf-mirror.com 社区提供的下载工具 hfd，它基于 aria2 多线程下载：

```bash
# 1. 下载并授权脚本
wget https://hf-mirror.com/hfd/hfd.sh
chmod a+x hfd.sh
# 2. 安装必要工具（如未安装）
sudo apt update && sudo apt install -y aria2 git-lfs
# 3. 下载特定数据集
./hfd.sh InternRobotics/InternData-N1 --dataset --hf_username bubblevan --hf_token ${YOUR_TOKEN} --include "vln_pe/traj_data/**" --tool aria2c -x 4
```
