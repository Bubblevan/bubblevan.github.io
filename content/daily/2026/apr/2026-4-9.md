先用`eval $(minikube docker-env -u)`切回本地 Docker Desktop 环境，然后了解一下K8S的几个基础概念：
可以把 K8s 想象成一个容器化应用的“操作系统”，它管理着许多机器（节点），而你通过 API 告诉它“我要跑什么、要几个、怎么连”。

1. Pod（豆荚）
- 最小、最基本的部署单元。一个 Pod 里可以有一个或多个容器（通常是 1 个）。
- 这些容器共享网络、存储、IP 地址，就像它们在同一台小虚拟机里。
- 类比：Pod 就像一艘小艇，里面的容器就是船员（通常一个船员就够了）。
- 例子：stablepay-api-gateway 跑在一个 Pod 里。

2. Deployment（部署）
- 管理 Pod 的“副本数”和“更新策略”。你告诉它“我要 3 个 Pod 一直运行”，它会保证始终有 3 个。
- 支持滚动更新、回滚、扩容缩容。
- 类比：Deployment 就像一个调度经理，确保你的服务有足够的小艇在跑，坏了的就换新的。

3. Service（服务）
- 给一组 Pod 提供一个稳定的访问入口。因为 Pod 的 IP 会变化（重启就变了），Service 提供一个固定的 IP 和 DNS 名字。
- 类型：
  - ClusterIP（默认）：只在集群内部可访问。
  - NodePort：在节点上开一个端口，外部可访问。
  - LoadBalancer：用云厂商的负载均衡器（生产常用）。
- 类比：Service 就像是前台总机，你打电话给总机（固定号码），它会转接到任意一个正在工作的员工（Pod）。

4. Namespace（命名空间）
- 资源隔离。不同团队、不同环境（dev/test/prod）可以放在不同 Namespace 里，互不干扰。
- `-n stablepay`，就是指定 stablepay 这个命名空间。

5. ConfigMap & Secret
- 配置和敏感信息。
- ConfigMap：存普通配置（如数据库地址、日志级别）。
- Secret：存敏感信息（如密码、Token），内容会 base64 编码（但只是编码，不是加密）。
- 它们可以挂载到 Pod 里变成文件，或者变成环境变量。

6. Ingress（入口）
- HTTP/HTTPS 路由规则。把集群外部的请求（比如 api.stablepay.com/payment）转发到集群内部的 Service。
- 类比：Ingress 就像大楼的大门和门牌指引，告诉来访者“支付服务往左，查询服务往右”。

7. PersistentVolume (PV) & PersistentVolumeClaim (PVC)
- 存储管理。PV 是集群的存储资源（如一块 NFS 盘、云硬盘），PVC 是 Pod 提出的存储申请。
- 就像“仓库里有货（PV），你填单子申请（PVC），就能领到一块空间”。

在 Docker 驱动下，Minikube 集群跑在 Docker 容器内，宿主机无法直接访问它的 NodePort。minikube service 命令会自动建立一个从宿主机 127.0.0.1 的随机高位端口（如 60821）到集群内服务 NodePort（30080）的转发隧道。当然也可以这样搞端口转发`kubectl -n stablepay-dev port-forward service/stablepay-api-gateway 28080:8080`。


然后是Ingress：
ALB = Application Load Balancer（应用型负载均衡），是阿里云提供的七层负载均衡产品（类似 AWS 的 ALB）。
在 Kubernetes 中，ALB Ingress Controller 会监听 Ingress 资源的创建，然后自动在阿里云上创建 ALB 实例，并将外部 HTTP/HTTPS 流量路由到集群内的 Service。
- 优点：阿里云原生，性能高，支持 HTTPS、域名转发、WebSocket 等。
- 缺点：需要额外安装 Controller（ACK 可选组件），并且会创建真正的云资源（会产生费用）。
保守一点使用开源的 Nginx Ingress Controller（也是 K8s 社区的标准方案）。它同样功能强大，且不绑定云厂商，配置迁移方便。
然后又遇到一个新错误：
```bash
The Service "stablepay-api-gateway" is invalid: spec.ports[0].nodePort: Forbidden: may not be used when 'type' is 'ClusterIP'
```
意思是：你的 stablepay-api-gateway Service 的 type 是 ClusterIP（默认），但你在 ports 里又写了 nodePort: 30080。在 K8s 中，nodePort 只能在 type: NodePort 或 type: LoadBalancer 时使用。这里我们把`services.yaml`里的`nodeport`换成ClusterIP算了，不用nodePort。
### 本科毕设
明天去跑A5的实验。有一说一A4已经证明在 DualVLN 的两阶段 S2 决策中，history 对 gateway 决策的作用具有时间分工，而不是简单的 recency bias；并且这种分工在多种弱视觉扰动下仍能复现。
如果只是“最近更重要”，那么你应该看到：
- keep_one：最近最好
- mask_one：最近最脆弱
但现在看到的是相反的分工：
- keep_one：最近最好
- mask_one：最早几帧更脆弱
这说明模型对 history 的利用不是单一时间衰减，而更像是角色分工：
- 最近 history 更像“我现在正看着哪儿、姿态如何、局部几何是否值得继续 lookdown”
- 早期 history 更像“全局走向 / 先前语境 / 轨迹一致性”的补充约束
当然，这还是推测层的解释，不是已经被证明的机制。但至少它已经是个有形状的现象了。
