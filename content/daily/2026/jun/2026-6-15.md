#### 权威 math / coding 数据集
**适合 SFT / GRPO 训练的数据：**

| 数据集 | 用途 | 备注 |
| :--- | :--- | :--- |
| **MATH** | SFT / eval 都可，但最好切分干净 | 12,500 道竞赛数学题，每题有 step-by-step solution，是 GSM8K 之后最经典的数学推理数据集之一，详细介绍见其[论文](https://arxiv.org/abs/2103.03874)。 |
| **NuminaMath-CoT / TIR** | 强 SFT，很适合数学 CoT / tool-integrated reasoning | [NuminaMath-CoT](https://huggingface.co/AI-MO/NuminaMath-CoT) 提供了 860k+ 数学竞赛题解对；[NuminaMath-TIR](https://huggingface.co/AI-MO/NuminaMath-TIR) 提供了 70k+ tool-integrated reasoning 解法。 |
| **OpenMathInstruct-1 / 2** | 强 SFT，尤其适合开源复现 | OpenMathInstruct-1 是 1.8M 数学 instruction tuning 数据；OpenMathInstruct-2 扩到 14M question-solution pairs，构建细节与开源动机可参阅[相关论文](https://arxiv.org/abs/2402.10176)。 |
| **MathInstruct / MAmmoTH** | 数学泛化 SFT | MathInstruct 混合了 CoT 和 PoT rationales，覆盖多个数学数据集，其设计思路来自 [MAmmoTH 论文](https://arxiv.org/abs/2309.05653)。 |
| **GSM8K** | 起步、sanity check、简单 RL reward | 包含 8.5K 小学数学应用题，其中 7.5K 训练、1K 测试，数据集细节与用法可参考 [GitHub 仓库](https://github.com/openai/grade-school-math)，适合先把 pipeline 跑通。 |

**更适合评测/holdout，不建议直接拿来训练污染的：**

| 基准 | 用途 |
| :--- | :--- |

| **Minerva Math / GaoKao / CollegeMath** | OOD math eval，可以看模型是否只会 GSM8K/MATH 模板。 |

#### 权威读物
按这个顺序读：
1. **DeepSeek-R1**：理解 RLVR / long-CoT / cold-start / distillation 总范式。
2. **DAPO**：最适合复现实验，因为它强调开源 recipe 和系统细节。
3. **VAPO / Dr. GRPO**：理解 GRPO/PPO 变体和长序列 credit assignment。
4. **ToRL 或 ReTool**：如果你要做 math + code interpreter/tool-use。
5. **SWE-RL / SkyRL-Agent**：如果你要做 coding agent 或 SWE-bench。
6. **Agent-R1 / RAGEN**：如果要做多轮 agent RL。
7. **LIMO / AgenticMath**：如果资源有限，走高质量数据/SFT 路线。
8. **Qwen3 / Qwen3-Coder-Next**：看工业级模型如何把 reasoning、coding、agent 统一到训练 recipe 里。

#### 研究热点
按“直接改进训练、数据、RL、tool-use、agent/coding 能力”的标准筛出来；不含纯 benchmark 论文，除非它同时提出训练或系统方法。
##### A. 通用 reasoning / RL 后训练主线

1. **DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via Reinforcement Learning** — 2025
   2025 年最重要的 reasoning-RL 论文之一，[DeepSeek-R1 的 arXiv 版本](https://arxiv.org/abs/2501.12948) 提出 R1-Zero/R1，用大规模 RL 激发推理，自带冷启动、多阶段训练与蒸馏路线；Nature 版本也确认其“纯 RL 可激发自反、验证、动态策略适应”等推理模式。
2. **Kimi k1.5: Scaling Reinforcement Learning with LLMs** — 2025
   Moonshot 的 RL scaling 技术报告，[Kimi k1.5 论文](https://arxiv.org/abs/2501.12599) 重点在多模态 LLM 的 RL 训练、数据 recipe、基础设施优化，是 R1 同期另一条工业级 RL scaling 路线。
3. **DAPO: An Open-Source LLM Reinforcement Learning System at Scale** — 2025
   [DAPO 工作](https://arxiv.org/abs/2503.14476) 提出 Decoupled Clip、Dynamic Sampling 等技巧，并开源训练代码/数据/系统，在 Qwen2.5-32B 上 AIME 2024 达到 50 分，重点是让 R1-style RL 可复现。
4. **VAPO: Efficient and Reliable Reinforcement Learning for Advanced Reasoning Tasks** — 2025
   [VAPO](https://arxiv.org/abs/2504.05118) 从 value-based PPO 路线修复 long-CoT RL 的 value bias、异构长度、稀疏奖励问题；报告在 Qwen-32B + AIME 2024 上达到 60.4。
5. **Open-Reasoner-Zero: An Open Source Approach to Scaling Up Reinforcement Learning on the Base Model** — 2025
   [Open-Reasoner-Zero](https://arxiv.org/abs/2503.24290) 目标是开源复现 R1-Zero 式训练，主张 vanilla PPO + rule-based reward + 简洁设置也能扩展 reasoning，并发布代码、参数、数据、权重。
6. **Understanding R1-Zero-Like Training: A Critical Perspective** — 2025
   [这篇批判性分析](https://arxiv.org/abs/2503.20783) 不是单纯提分论文，而是分析 R1-Zero-like 训练中 base model 与 RL 的作用，并指出预训练 bias 和 GRPO 优化选择会影响“aha moment”等现象。
7. **Advancing Language Model Reasoning through Reinforcement Learning and Inference Scaling / T1** — 2025
   THUDM 的 T1 路线在[该论文](https://arxiv.org/abs/2501.11651)中提出：用 trial-and-error、自验证 CoT 初始化，再用 RL 和 inference scaling 提升数学 reasoning。
8. **Qwen3 Technical Report** — 2025
   [Qwen3 技术报告](https://arxiv.org/abs/2505.09388) 把 thinking mode 与 non-thinking mode 统一，并引入 thinking budget；报告称在 code generation、math reasoning、agent tasks 等 benchmark 上有强结果。
9. **Revisiting Reinforcement Learning for LLM Reasoning from A Cross-Domain Perspective / Guru** — 2025
   [Guru 工作](https://arxiv.org/abs/2506.14965) 构建 92K 可验证 reasoning corpus，覆盖 Math、Code、Science、Logic、Simulation、Tabular，强调 RL 在不同领域的泛化与技能习得差异。
10. **Absolute Zero: Reinforced Self-play Reasoning with Zero Data** — 2025
    [Absolute Zero](https://arxiv.org/abs/2505.03335) 不依赖外部训练数据，让模型自己提出任务、自己解题，用 code executor 验证，形成 self-play curriculum；报告称在 coding 和 math reasoning 上超过其他 zero-setting 模型。
11. **R-Zero: Self-Evolving Reasoning LLM from Zero Data** — 2025
    [R-Zero](https://arxiv.org/abs/2508.05004) 采用 Challenger/Solver 双模型共演化，完全从零生成训练任务与标签，提升数学与通用 reasoning。
12. **LIMO: Less is More for Reasoning** — 2025
    不是 RL，而是极小规模高质量 SFT 路线：[LIMO](https://arxiv.org/abs/2502.03387) 用 817 条 curated 样本显著提升数学推理，提出“Less-Is-More Reasoning Hypothesis”。

##### B. Math reasoning / tool-use / code-interpreter 方向

13. **ToRL: Scaling Tool-Integrated RL** — 2025
    [ToRL](https://arxiv.org/abs/2503.23383) 训练 LLM 自主使用计算工具；ToRL-7B 在 AIME 2024 达到 43.3%，比无工具 RL 和已有 TIR 模型更强。
14. **ReTool: Reinforcement Learning for Strategic Tool Use in LLMs** — 2025
    [ReTool](https://arxiv.org/abs/2504.11536) 用 RL 学“何时、如何调用 code interpreter”，在 AIME 上显著优于纯文本 RL baseline，并出现代码自修正行为。
15. **THOR: Tool-Integrated Hierarchical Optimization via RL for Mathematical Reasoning** — 2025 / ICLR 2026
    [THOR](https://arxiv.org/abs/2509.13761) 提出 TIRGen、多智能体 actor-critic 数据构造、trajectory-level + step-level 分层 RL，以及基于工具反馈的 self-correction；ICLR 2026 页面显示该论文条目。
16. **Tool-Star: Empowering LLM-Brained Multi-Tool Reasoner via Reinforcement Learning** — 2025
    [Tool-Star](https://arxiv.org/abs/2505.16410) 是多工具 reasoning 框架，结合 tool-use 数据合成、cold-start tuning 和 hierarchical reward 的 self-critic RL。
17. **AgenticMath: Enhancing LLM Reasoning via Agentic-based Math Data Generation** — 2025
    [AgenticMath](https://arxiv.org/abs/2510.19361) 用多智能体流程生成/改写/增强数学 QA 数据，再用于 SFT，小模型用 30K–60K 高质量样本获得竞争性数学推理表现。
18. **AgentMath: Empowering Mathematical Reasoning for LLMs via Tool-Augmented Agent** — 2025
    [AgentMath](https://arxiv.org/abs/2512.20745) 把自然语言 CoT 转成 tool-augmented trajectories，并用 agentic RL 学习多轮代码执行和工具使用策略；报告在 AIME24/AIME25/HMMT25 上有很强结果。
19. **ReasonFlux: Hierarchical LLM Reasoning via Scaling Thought Templates** — 2025
    [ReasonFlux](https://arxiv.org/abs/2502.06772) 通过 thought template library + hierarchical RL + inference scaling 提升数学推理；报告称 ReasonFlux-32B 在 MATH/AIME 上超过 o1-preview 和 DeepSeek-V3。
20. **Scaf-GRPO: Scaffolded Group Relative Policy Optimization** — 2025
    [Scaf-GRPO](https://arxiv.org/abs/2510.19807) 针对数学 reasoning 改进 GRPO，报告称 Qwen2.5-Math-7B 在 AIME24 上 pass@1 相对 vanilla GRPO 提升 44.3%。

##### C. Coding / software engineering agent 方向

21. **SWE-RL: Advancing LLM Reasoning via Reinforcement Learning on Open Software Evolution** — 2025
    [SWE-RL](https://arxiv.org/abs/2502.18449) 把 RL 用到真实软件演化数据和 GitHub issue 修复；Llama3-SWE-RL-70B 在 SWE-bench Verified 上达到 41.0%，并在代码、数学等域外任务出现泛化提升。
22. **DeepCoder: A Fully Open-Source 14B Coder at O3-mini Level** — 2025
    Agentica + Together AI 的开源 coding RL 模型，详见 [DeepCoder 官方博客](https://www.together.ai/blog/deepcoder)，使用 24K 可验证 coding problems 训练，报告 LiveCodeBench Pass@1 60.6%，接近/超过 o3-mini 低档。
23. **Competitive Programming with Large Reasoning Models** — 2025
    OpenAI 在[这篇论文](https://arxiv.org/abs/2502.06807)中展示 RL 显著提升复杂编程与 reasoning；比较 o1、早期 o3、o1-ioi，指出通用 RL scaling 比手工竞赛策略更强。
24. **rStar-Coder: Scaling Competitive Code Reasoning with a Large-Scale Verified Dataset** — 2025
    [rStar-Coder](https://arxiv.org/abs/2505.21297) 构造 418K 竞赛级代码题、580K 长推理解法和可验证测试；Qwen2.5-14B 在 LiveCodeBench 上从 23.3% 提升到 62.5%。
25. **ReST-RL: Achieving Accurate Code Reasoning of LLMs with Reinforcement Learning** — 2025
    [ReST-RL](https://arxiv.org/abs/2508.19576) 结合改进 GRPO 和 value-model-assisted test-time decoding，用于提升 code reasoning。
26. **Teaching LLM to Reason: Reinforcement Learning from Algorithmic Problems without Code / TeaR** — 2025
    [TeaR](https://arxiv.org/abs/2507.07498) 用 algorithmic problems 引导模型学习代码相关的推理结构，但避免直接过拟合复杂代码模式；在 17 个 Math/Knowledge/Code/Logic benchmark 上提升。
27. **Agent-R1: Training Powerful LLM Agents with End-to-End Reinforcement Learning** — 2025
    [Agent-R1](https://arxiv.org/abs/2511.14460) 为 LLM agent 定义 MDP 组件，并提出训练框架，面向多轮交互和工具使用。
28. **RAGEN: Understanding Self-Evolution in LLM Agents via Multi-Turn Reinforcement Learning** — 2025
    [RAGEN](https://arxiv.org/abs/2504.20073) 提出 StarPO 和系统，专门研究多轮 agent RL 训练中的长程决策、随机环境反馈、reward collapse 等问题。
29. **SkyRL-Agent: Efficient RL Training for Multi-turn LLM Agent** — 2025
    [SkyRL-Agent](https://arxiv.org/abs/2511.16108) 针对 SWE-Bench 等长程真实环境任务做多轮 agent RL；报告训练 SA-SWE-32B，并用异步 pipeline 与 AST search tool 提升训练效率。
30. **Qwen3-Coder-Next Technical Report** — 2026
    [Qwen3-Coder-Next](https://arxiv.org/abs/2603.00729) 是面向 coding agents 的开源权重模型，80B 总参数、推理激活 3B；通过可验证 coding task 合成、可执行环境、中期训练和 RL 从环境反馈中学习。
31. **GLM-5: from Vibe Coding to Agentic Engineering** — 2026
    [GLM-5](https://arxiv.org/abs/2602.15763) 强调 agentic、reasoning、coding 能力，目标从 vibe coding 转向 agentic engineering。
32. **ReaComp: Compiling LLM Reasoning into Symbolic Solvers for Efficient Program Synthesis** — 2026
    [ReaComp](https://arxiv.org/abs/2605.05485) 用 coding agents 把 reasoning traces 编译成可复用 symbolic solvers，在 program synthesis 上降低 token 成本并提升 hard benchmark。
