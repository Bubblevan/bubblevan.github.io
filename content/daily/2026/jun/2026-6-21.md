## 今日总结
### 面向AI编程Agent的结构化学习工作流技能
```bash
npx skills add mattpocock/skills --skill teach
```
- `skills`：轻量CLI工具 `@jeril/skills_cli`，专门用来给AI编程Agent（Claude Code、Cursor、OpenClaw、Codex等）安装标准化技能（Skill），不用全局安装，`npx` 临时拉起运行。
- `add mattpocock/skills`：指定从 GitHub 仓库 `mattpocock/skills`（TS大神Matt Pocock开源的AI工程化技能合集）拉取技能包。
- `--skill teach`：**只单独安装其中名为 `teach` 的技能**，而非全量安装仓库里所有工具。

`teach` 核心作用：把项目目录变成可持久化、有状态的学习空间，让AI跟着一套标准化流程做知识点学习、沉淀笔记、记录掌握进度。
底层遵循 Claude Agent Skill 规范，本质是一套带配套模板文件的标准化Prompt工作流。

#### 执行这条命令会产生什么结果
##### 1. 文件层面（项目根目录自动生成）
1. 在项目 `.claude/skills/` 目录复制 `teach` 完整技能文件（SKILL.md执行指令、配套模板、子文档），Agent会话内输入 `/teach` 即可调用该能力。
2. 自动生成一套学习目录结构：
    - `MISSION.md`：当前学习目标、要掌握的知识点清单
    - `lessons/`：分章节课程、实操练习文档
    - `learning-records/`：学习进度、已掌握/未掌握知识点记录
    - `reference/`：自动生成的速查手册、代码示例、知识点总结
3. 所有学习状态**全部以本地文件存储**，不依赖云端数据库，可提交Git、跨环境复用。

##### 2. Agent使用层面（Claude Code / OpenClaw / Cursor 生效）
1. 会话内输入 `/teach` 触发交互式学习流程：AI会分步引导你定义学习目标、拆解课程、做实操练习、验收掌握程度。
2. 支持持久化学习上下文：关闭会话后重新打开，AI读取本地 `learning-records` 自动恢复上次学习进度，不会丢失记忆。
3. 配套能力：学习完成后自动输出结构化知识总结、可复用代码片段，方便后续开发查阅。

##### 3. 附加特性（teach专属设计）
- 支持脱敏沉淀代码、知识点，适合学习TS、区块链、AI Agent、支付SDK这类复杂技术栈；
- 学习过程全程留痕，每一步练习、思考、错题都存在本地文件，可追溯复盘；
- 和 mattpocock 其他技能（`grill-me`、`tdd`、`diagnose`）互通，可搭配做“先学习→再实操编码”完整流程。

如果执行 `npx skills add mattpocock/skills`（不带参数），会弹出多选框让你批量安装全部技能；
加 `--skill teach` 是**精准只安装这一个学习技能**，轻量化、不引入无关工具，适合只需要AI辅助学习的场景。

2份PRD/TRD，分别对应CLI/MCP的创建（这还要加一个PROGRESS），和x402 SDK的计划迁移（payment_event相同）

#### x402 协议
x402 由 x402 基金会官方维护 SDK，核心语言均为第一方实现，是协议的标准参考实现，完全对齐最新协议规范。

| 语言 | 官方包/仓库 | 成熟度 | 核心能力 |
|------|-------------|--------|----------|
| TypeScript/Node.js | `@x402/client`、`@x402/fetch`、`@x402/axios` 及各链适配包（`@x402/evm`、`@x402/svm` 等） | 最高（首发实现） | 完整客户端、服务端中间件、全链支付支持 |
| Go | `github.com/x402-foundation/x402/go` | 高 | 客户端、服务端中间件、核心支付逻辑 |
| Python | `x402` PyPI 包（支持 httpx/requests 两种 HTTP 客户端） | 高 | 客户端集成、FastAPI 服务端中间件 |

- 官方 SDK 同时覆盖**买方（客户端）**和**卖方（服务端）**两侧：客户端自动处理 402 状态码、支付签名、请求重试；服务端提供 Express/FastAPI 中间件，一键开启接口计费。
- 支持按次计费、批量结算、托管充值等多种支付方案，所有方案均在官方 SDK 中原生实现，无需自行封装。
- Rust、Java 等其他语言目前只有社区实现，暂无官方原生 SDK。


#### MCP（Model Context Protocol）
MCP 由 Anthropic 发起并主导，官方组织 `modelcontextprotocol` 直接维护多语言 SDK，所有官方实现严格对齐协议规范，是各类 AI 客户端（OpenClaw、Claude Code、Cursor 等）的兼容基准。

| 语言 | 官方仓库/包 | 发布状态 |
|------|-------------|----------|
| TypeScript/Node.js | `@modelcontextprotocol/sdk` | 稳定首发，生态最完善 |
| Python | `mcp` PyPI 包 | 稳定首发，官方原生维护 |
| Go | `modelcontextprotocol/go-sdk` | 正式稳定版（Google Go 团队协作开发） |
| C#/.NET | `modelcontextprotocol/csharp-sdk` | 稳定 v1.0（与微软联合开发） |
| Java | 官方 Java SDK | 正式发布，支持 Spring 生态适配 |

- 所有官方 SDK 均同时支持**客户端**和**服务端**开发，原生兼容 stdio、HTTP SSE 两种主流传输方式，覆盖工具、资源、提示词等全部 MCP 核心能力。
- 官方提供大量参考服务器实现（GitHub、文件系统、数据库等），均可直接复用。
- 其余语言（Rust、PHP 等）为社区第三方实现，兼容性与更新节奏不保证与官方同步。
