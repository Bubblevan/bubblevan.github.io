还有页眉页脚、分页符分节符的知识。WPS 用 **"节"** 来控制独立排版，每节可以有自己的页眉页脚。默认所有页面属于同一节，修改一个会影响全部。实现指定范围的奇偶页页眉，必须：
- 用分节符把文档切成 "不需要页眉" 和 "需要页眉" 两个独立部分
- 断开新节与前一节的页眉链接（否则修改会同步到前面）
- 在新节内启用 "奇偶页不同"，分别设置内容和对齐方式

还有英文与字母（以及参考文献上标里）改 Times New Roman的方法：
1.  按 `Ctrl + H` 打开「查找和替换」对话框
2.  点击左下角**更多**，勾选**使用通配符**（必须勾选！）
3.  **修改查找内容**为：`[\[\]a-zA-Z0-9]`
    - 关键说明：`\[` 和 `\]` 是转义后的中括号（因为中括号本身是通配符，需要加反斜杠表示"我要找的就是括号本身"）
    - 这个表达式会同时匹配：左中括号`[`、右中括号`]`、所有大小写字母、所有数字
4.  光标移到「替换为」框（**什么都不要输入**）
5.  点击**格式**→**字体**，西文字体选择 `Times New Roman`，点击确定
6.  点击**全部替换**

以及 **GB/T 7714标准要求的悬挂缩进2字符**（第一行[1]顶格，第二行及以后所有文字缩进2字符，与第一行的文字开头对齐）：
1.  **全选所有参考文献内容**（从[1]一直选到最后一条的末尾）
2.  右键点击选中的内容 → 选择**「段落」**
3.  在弹出的段落设置窗口中：
    - 找到「特殊格式」下拉框，选择**「悬挂缩进」**
    - 缩进值设置为**「2字符」**（这是国家标准，刚好和[1]加一个空格的宽度对齐）
    - 点击**确定**，完成！

### TypeScript 必须编译成 JavaScript
TS 确实比 JS 更高级，类型安全是它最大的优势，但TS 从设计之初就不是用来替代 JS 的，而是 JS 的 "开发时增强工具"，最终必须编译成纯 JS 才能运行。

| 特性 | 详细说明 |
| :--- | :--- |
| **1. 所有运行环境只认识 JavaScript** | 浏览器（V8 引擎）、Node.js、Electron 等运行环境原生只支持执行 JavaScript 代码，完全不识别 TypeScript 语法（如 `number` 类型注解、`interface` 接口、`enum` 枚举等）。编译过程会**擦除**所有 TS 特有语法，只保留标准 JS 代码。 |
| **2. TS 的类型安全只在编译时生效** | 类型检查仅在编译时进行一次，编译完成后所有类型信息被**完全删除**，生成的 JS 代码与手写 JS 没有任何区别。**运行时不存在类型检查**，因此 TS 项目仍可能出现类型错误（如外部传入的数据不符合预期）。 |
| **3. 完美的向下兼容性** | TS 编译器（`tsc`）可将最新的 TS 代码编译成任意旧版 JS（从 ES3 到 ESNext）。例如，`async/await` 会被自动编译成 ES5 时代的 Promise 回调写法，使现代代码能在 IE6 等古老浏览器中运行，解决了 JS 生态最头疼的兼容性问题。 |
| **4. 零运行时开销** | 由于编译时删除了所有类型信息，生成的 JS 文件大小与手写 JS 几乎相同，运行速度与原生 JS 完全一致，没有任何性能损失。 |
