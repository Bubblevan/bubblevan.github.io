### 代码技巧
#### Github Fork
学到了fork的新用法：添加一个新的 remote 指向你的 fork，保留原仓库为 upstream
```bash
git remote rename origin upstream
git remote add origin https://github.com/你的用户名/vllm_dualvln.git
```
之后可以通过 git push origin main 推送到我的 fork，通过 git fetch upstream 获取原仓库更新
#### WSL 走代理
一般来说它会`wsl: 检测到 localhost 代理配置，但未镜像到 WSL。NAT 模式下的 WSL 不支持 localhost 代理。`
WSL 默认是 NAT；在 Windows 11 22H2 及以上，可以启用 mirrored networking，兼容性更好，也更适合这类代理/网络问题
因此只要往`C:\Users\你的用户名\.wslconfig`里添加下面的内容就可以了：
```
[wsl2]
networkingMode=mirrored
dnsTunneling=true
firewall=true
autoProxy=true
```
然后管理员powershell执行`wsl --shutdown`再重新打开ubuntu（这里Docker Desktop竟然崩溃了啊，提醒我们不要开着这个的时候shutdown wsl）
`curl -I https://solana-install.solana.workers.dev`这个时候就能返回`HTTP/1.1 200 Connection established`了。

```bash
Creating token HQx6BQtQb5cqGEyFofQw3acKz9KvL8xzPy6riE6k5uDF under program TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA

```
这个代币的基础属性就被定义好了：
- 这是一个新代币
- 小数位是 6
- 谁拥有铸币权限(Mint Authority)

接下来给自己创建一个持币账户：`spl-token create-account <TOKEN_MINT>`
```bash
(base) bubblevan@Bubbles:~$ spl-token create-account HQx6BQtQb5cqGEyFofQw3acKz9KvL8xzPy6riE6k5uDF
Creating account 2bDvNtPtcwjPdpjTSwhvfxrHjna9EbK3Tmoa8erQGRHg

Signature: Lqycx3VqYRXtxtyWpGTA12fobCacYDp6w7L2L64o3a8dgM4t78KX4FgjszXM5V763BVrQcev8qq6JbtnVimgADE
```
这一步不是再创建一个代币，而是创建一个**用来存这个代币余额的账户**。可以这么理解：
- create-token = 发行“币种”
- create-account = 给你的钱包开一个“这个币的余额账户”
因为在 Solana 上，代币本体和你持有余额的账户是分开的。

随后铸造 1000000 个代币到我的账户：`spl-token mint <TOKEN_MINT> 1000000`
