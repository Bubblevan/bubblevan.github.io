### [Humanify实习招募](https://humanify.io/)
这个实习大概率不是做应用层套壳**，而是偏“**基座/后训练 + 数据评测 + 系统化落地**”的科研工程混合岗。

- 他们对外叙事是：做 **human-like 模型 + AI 原生 OS**，把“自主认知/人格/情商”做成 OS 级能力（而不是单纯做一个 Agent 应用）。
- 在[创始人访谈](https://zhuanlan.zhihu.com/p/1994835220399159168?utm_source=chatgpt.com)里讲得更直白：**“模型就是操作系统，操作系统就是模型”**，希望系统能在没有明确指令下感知场域、形成动机，并长期建立对人的稳定认知。 
- 技术侧重点里反复出现两件事： 

  1. **实时/always-on** 的自主系统（不是“你问一句它回一句”）
  2. **上下文与长期记忆管线**（用户画像、记忆整合、减少遗忘）
- 语音被他们当作“最自然交互方式”，会投入很多在语音侧。
- Hugging Face 上他们以 Humanify Inc. 名义有组织页，并且近期在更新 **speaker / speech / evaluation** 相关数据集（能侧面印证“语音侧在做事”）。 
- 工商层面公司实体是“人格智能（杭州）有限公司”，公开工商信息显示成立日期为 2025-06-06 

这个“科研实习生”最可能做什么（按概率从高到低） :

#### 1) 前沿论文复现 + 快速实验验证（高概率） 

* 复现 LLM/多模态/Agent/World Model 论文的**关键结论或关键模块**，把“论文里说的提升”在你们数据/设置上跑通（JD 明确写了复现与实验对比）。
* 输出通常是：复现代码、实验日志、对比表、失败原因分析、下一步改进方案。

#### 2) “类人认知/自主性”的后训练与数据构建（高概率） 

他们访谈里明确说小公司做不了大规模预训练，但能**定义任务、做后训练**，并且已经在“泛化模态上做进一步训练让认知涌现”。
你在这里可能做：

* 设计/整理用于“主动性、情绪理解、场域理解、社交破冰”等能力的数据（合成 + 人工标注 + 规则弱监督都可能）。
* 做 SFT / DPO / RM / RLHF 之类的后训练实验（具体方法取决于团队路线）。
* 做“像人”的评测：例如**情绪细粒度识别、回应的合适度、主动介入时机**等（他们强调“恐怖谷”、强调更细微情绪捕捉）。

#### 3) 长期记忆/用户画像/上下文工程（高概率） 

他们把“长期记忆与理解”直接当作自主系统的关键前提，并提到“记忆管线”。
对应实习内容可能是：

* 记忆写入/检索/压缩（summarize）、冲突消解、时间衰减、重要性打分。
* 记忆一致性评测：同一用户多轮对话后是否“忘事”、是否“画像漂移”。
* 记忆与模型结合：纯 RAG 式记忆、结构化 memory、或训练时注入的 memory token/adapter（不确定，但方向会围绕“减少遗忘、增强长期一致性”）。

#### 4) 语音/说话人/音频鲁棒性与评测（中高概率） 

* 他们公开强调语音交互；HF 上又能看到 speaker/speech/evaluation 数据集。
  你可能做：
* 语音相关数据清洗、切分、对齐（音频+文本）、说话人/情绪标签处理。
* 说话人相关评测集与指标（例如 speaker evaluation 的测试集、不同噪声/场景鲁棒性）。
* 语音链路的工程实验：流式推理、VAD、端侧/低延迟方案（具体看团队资源）。

#### 5) 工程落地与工具链（中概率，但很可能“顺带做”） 

即使你是科研实习，JD 也写了“工程落地”。他们在其他招聘里还提到 LLMOps、推理加速、评测、RAG、Memory 等工具链建设。
所以你可能会被期待：

* 把训练/评测脚本工程化：可复现、可配置、可追踪（wandb/mlflow 之类）。
* 做推理与评测流水线：batch 跑分、自动出报表、ablation 管理。
* 做 demo/服务封装，让模型能力能被产品/OS 层调用。


最常见的“可交付物”长什么样 :

* **一份复现报告**：复现设置、关键超参、对照组、结论是否成立、偏差原因。
* **一套数据/评测基准**：数据 schema、采样策略、指标、leaderboard（内部的也算）。
* **一组后训练实验结果**：曲线、消融、失败案例分析、下一轮实验计划。
* **论文/技术文档**：他们明确说要做论文投稿（但是否真投、投什么会取决于团队节奏）。
