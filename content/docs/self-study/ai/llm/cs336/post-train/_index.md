## 从 Pre-train 到 Reasoning RL
```
原始海量文本
   ↓
[Pre-training]
学语言分布 / 世界知识 / 基础模式
   ↓
Base Model
   ↓
[Continue Pre-training]
补某个领域的数据密度（数学/代码/法律/医学）
   ↓
Domain Base Model
   ↓
[SFT]
学会“看懂指令 → 按格式回答”
   ↓
Instruct Model
   ↓
[PPO / DPO]
学人类偏好：更有用、更安全、更像人想要的回答
   ↓
Aligned Model
   ↓
[Reasoning RL / GRPO]
针对有客观标准答案的任务，在线搜索更好的推理路径
   ↓
Reasoning Model
```

如上所示，首先的基础是预训练。**预训练不是在教模型做题，而是在教模型“这个世界上的 token 通常如何一起出现”**。语法、常识、事实共现、文体模式，甚至一部分推理雏形，本质上都能在大规模文本压缩里冒出来。

$$
p(x_t \mid x_{<t})
$$

只要数据足够大、模型足够强，这个目标就会逼它把“前文里哪些模式对后文最有信息量”都编码进参数里。于是语法、事实关联、风格、局部规划能力，都会以“有利于预测下一个 token”的形式被学进去。

InstructGPT 从这个前提出发：纯预训练模型很强，但**并不会天然按用户意图工作**。它只知道“**接下来什么最像互联网上会出现的文本**”，不知道“**用户现在是在给我下指令**”。

你问它“如何制作蛋糕？”，它可能写菜谱，也可能写蛋糕历史；你问它“求 (x^2) 的导数”，它也可能顺手展开成一段教材体解释。不是它没见过，而是它**没有被训练成优先执行指令**。

预训练阶段学到的是：

* 语言基本规律
* 世界知识的广泛覆盖
* 推理的“雏形”，但不是受控推理

它没学到的是：

* 指令边界
* 任务优先级
* 特定领域的高密度精细知识

这就自然引出下一步：**继续预训练**。

这里你以为我要说 SFT？但是 **SFT 改的是行为策略，Continue Pre-train 改的是底层分布**。

如果某个领域（数学、代码、法律）的 token 组合、专业术语、证明风格、公式变换，在通用语料里占比极低，那么模型虽然“听说过”，但参数里这块区域是稀的、浅的、不稳的。直接 SFT，它会更像背出一套答题姿势。这个阶段的目标，其实还是老目标 —— 损失函数通常不变，还是 next-token prediction。

也就是说，我们不是在改“任务形式”，而是在改“训练数据分布”。

- **Pre-train / Continue Pre-train** 的核心区别不在 loss，而在 data distribution。
- 这个阶段训练出来的仍然是 **Base-like model**，不是天然能聊天的 Instruct Model。

Continue Pre-train 解决的是：

* 专业词汇稀疏
* 领域逻辑分布太浅
* 通用模型在理科任务上“浅尝辄止”

但它仍然没解决：

* 不懂指令
* 不会按人类偏好表达
* 不会稳定地产生 CoT 风格输出

所以终于，进入我们的 **Post-train** 环节。
