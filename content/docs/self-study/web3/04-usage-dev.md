主要就是设置钱包、发送交易、探索区块链。
你的钱包管理的是你的密钥，而不是你的钱。你的密钥控制着你的 Solana 账户。交易用来转移 SOL 和与程序交互。区块浏览器可以查看所有链上数据。

## 钱包
Solana 钱包并不存储你的加密货币——币本身作为账本条目存在于区块链上。钱包存储的是你的私钥，并生成数字签名来证明你有权花费这些币。这里用Phantom就行。
转这种货币需要在某些交易所去转。

我是拿到了12个助记词，这玩意是你的私钥的人类可读形式。任何人拿到你的助记词就能控制你的所有账户和资产。

## SOL
使用 Solana 需要支付交易手续费，所以你需要持有一些 SOL。

### 主网（真金白银的 SOL）
- 中心化交易所：在 Coinbase、Binance、Kraken 等平台购买 SOL，然后提现到你的钱包地址。
- 去中心化交易所：在 Jupiter、Raydium、Orca 等平台用其他加密货币兑换 SOL。
- 法币入口：使用 Moonpay、Transak 等服务直接用信用卡购买 SOL。

### 开发网（测试用的 SOL）
如果你想在不动用真钱的情况下测试，可以使用 devnet：
```bash
# 安装 Solana CLI
sh -c "$(curl -sSfL https://release.solana.com/stable/install)"

# 配置为开发网
solana config set --url https://api.devnet.solana.com

# 请求空投（测试 SOL）
solana airdrop 2

# 查看余额
solana balance
```

## 发送一笔交易
下面以 Phantom 钱包为例，演示如何发送 SOL。

**步骤 1**：打开 Phantom，确保你有 SOL  
检查余额。你需要至少有 0.000005 SOL 作为手续费，加上你要发送的金额。

**步骤 2**：点击“Send”  
输入接收方的 Solana 地址（格式如 `7TpX...9kLM`，43 个 base58 编码字符）或扫描其二维码。

**步骤 3**：输入金额  
指定要发送多少 SOL。钱包会显示交易手续费（通常为 0.000005 SOL）。

**步骤 4**：确认并发送  
仔细检查：
- 接收方地址是否正确（发错地址无法撤回）
- 金额是否正确
- 你的余额是否足够支付“金额 + 手续费”

确认无误后点击确认。你的钱包会用私钥签署这笔交易。

**步骤 5**：交易处理  
钱包将签名的交易广播到网络，你会看到状态从“Pending”（等待中）变为“Confirmed”（已确认），通常只需 1-2 秒。

- 你的钱包构建了一条包含系统程序转账指令的交易。
- 交易声明了要访问的账户（你的账户和接收方账户）。
- 钱包用你的私钥签名。
- 将签名后的交易发送给 RPC 节点。
- RPC 节点转发给当前的区块生产者（Leader）。
- Leader 将交易打包进区块。
- 验证节点确认该区块。
- 交易成为最终状态，不可逆转。

## 区块浏览器
区块浏览器让你查看所有链上数据。Solana 上的一切都是公开且可验证的。

[Solana Explorer（官方）](https://explorer.solana.com)可以查看：
- 按交易签名查询交易
- 按地址查询账户
- 按 slot 编号查询区块
- 程序及其交互

使用方法：在搜索框输入交易签名、账户地址或区块 slot，即可看到完整详情。  
对于交易，你可以看到：
- 执行的所有指令
- 访问了哪些账户
- 调用了哪些程序
- 成功还是失败
- 消耗的计算单元
- 支付的手续费

对于账户：
- 当前 lamports 余额
- 所有者程序
- 数据大小和租金状态
- 完整交易历史
- 持有的代币（如果有）

[Solscan](https://solscan.io)界面更友好，代币展示更清晰，有 NFT 图库和分析功能，非技术用户更喜欢用它。

## 理解交易签名
每笔交易都有一个唯一的签名——一个 88 个字符的 base58 字符串，例如：
```bash
3nGq2yczqCpm8bF2dyvdPtXpnFLJ1gKWyPp4EYFS8k8wHqJLfcBBdpKYmKCKMJhpYcXqmLWdWcNQjTiJhHWMVdU5
```
这个签名就是你的收据。有了它，你可以：
- 在区块浏览器上查看这笔交易
- 证明交易确实发生过
- 验证交易的详细信息

发送交易后，钱包会显示这个签名。复制它并到区块浏览器搜索，就能看到当时发生的所有细节。

- **常见的交易结果**
  - **Success（成功）**：交易执行，状态更新，显示“Success”。
  - **Failure（失败）**：交易被打包进区块但由于程序错误被回滚。你仍然需要支付手续费。
  - **Dropped（丢弃）**：交易未能在超时时间内被打包，没有支付手续费，需要重试。
