## Leetcode 题型汇总
| 数据结构       | 应用场景       | 难度   | 典型题目（LeetCode）                          | 核心思路                                                                 |
|----------------|----------------|--------|-----------------------------------------------|--------------------------------------------------------------------------|
| 队列 (Queue)   | 设计循环队列   | Medium | 622. Design Circular Queue                    | 使用固定大小的数组模拟循环队列，维护 front 和 rear 指针（或 head 和 tail），通过取模运算实现环式移动，注意判断空/满的条件（可用计数器或预留一个空位） |
|                | 任务调度器     | Medium | 621. Task Scheduler                           | 优先队列（最大堆）存储任务频次，每次弹出频次最高的任务执行，配合队列冷却等待；贪心选择频次最高的任务可最小化空闲时间 |
|                | 数据流移动平均值 | Easy/Medium | 346. Moving Average from Data Stream       | 使用队列（或双端队列）维护最近 size 个数值，保持窗口和，每次新值进入时更新窗口和并计算平均值 |
| 双端队列 (Deque)| 设计循环双端队列 | Medium | 641. Design Circular Deque                    | 类似循环队列，但需支持两端插入/删除；可用数组或双向链表实现，注意边界条件与空/满判断 |
|                | 滑动窗口最大值 | Hard   | 239. Sliding Window Maximum                   | 单调递减双端队列存储索引，队列头为当前窗口最大值。遍历时移除出窗口的索引，并保证队列内索引对应的值递减，每次输出队列头对应的值 |
|                | 和至少为 K 的最短子数组 | Hard | 862. Shortest Subarray with Sum at Least K | 前缀和 + 单调递增双端队列。维护前缀和数组，队列存储索引，保证前缀和递增；对于每个位置，从队头弹出满足 prefix[i] - prefix[queue.front()] >= K 的索引并更新最短长度 |
| 链表 (Linked List)| 合并K个排序链表 | Hard | 23. Merge k Sorted Lists                     | 优先队列（最小堆） 存储每个链表的当前节点，每次弹出最小值，将其下一个节点（如果存在）加入堆中，直到堆为空 |
|                | K 个一组翻转链表 | Hard   | 25. Reverse Nodes in k-Group                  | 递归或迭代。每 k 个节点为一组，断开、反转、再连接；注意剩余节点不足 k 个时保持原状，需精确操作指针 |
|                | 复制带随机指针的链表 | Medium | 138. Copy List with Random Pointer          | 哈希表 存储原节点到拷贝节点的映射；先遍历创建所有节点并建立映射，再遍历设置 next 和 random 指针。或使用“交错插入”法（O(1) 空间） |
|                | LRU 缓存       | Medium | 146. LRU Cache                                | 哈希表 + 双向链表。哈希表快速定位节点，双向链表维护访问顺序（最近使用的在头部，最久未使用的在尾部）。get 和 put 操作均需 O(1) 时间，注意容量满时淘汰尾部节点 |
|                | LFU 缓存       | Hard   | 460. LFU Cache                                | 多层哈希表 + 双向链表。维护 key‑>node 映射、freq‑>链表 映射（每个频率一个双向链表，维护最近使用顺序），以及当前最小频率。get/put 时更新频率、调整链表，容量满时删除最小频率链表中最久未使用的节点 |
