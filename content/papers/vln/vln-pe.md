## 研究背景
VLN 任务经历三代发展，逐步向物理真实场景逼近：
阶段	技术特点	代表平台	局限性
VLN（2018）	基于预定义拓扑图的 “瞬移式” oracle 导航	MP3D simulator	忽略机器人物理运动，理想化定位
VLN-CE（2020）	连续导航（无拓扑图），伪机器人（圆柱代理）	Habitat	缺乏真实机器人动力学，未考虑物理约束
VLN-PE（2025）	物理真实导航，支持多类型实体机器人	GRUTopia+Isaac Sim	首次覆盖物理 / 视觉差异，贴近实际部署
![](/paper/vln-pe-develop.png)

- **模拟 - 物理鸿沟**：现有 VLN 模型基于理想化模拟训练，迁移到物理机器人时性能大幅下降（如碰撞、跌倒、视角偏移）。
- **实体多样性缺失**：多数基准仅支持轮式 / 点代理，忽略人形、四足机器人的运动约束（如腿部机器人易卡缝、跌倒）。
- **测试条件单一**：忽视光照变化、场景多样性（如 MP3D 场景视觉多样性有限），模型泛化能力差。
- **缺乏全栈平台**：无平台整合 “真实机器人动力学 + 精准运动控制 + 可扩展训练”，难以评估跨实体适应能力。

## 方法论
![](/paper/vln-pe-overview.png)

### 一、Simulations（仿真基础）：物理真实与多机器人兼容的底层支撑
VLN-PE的仿真层核心是解决传统VLN基准“伪动作执行”（依赖动画/预定义位置，无真实物理约束）的问题，构建贴近真实机器人部署的模拟环境，关键设计如下：
- **底层模拟器选型**：基于**GRUTopia**（依托NVIDIA Isaac Sim开发），具备高保真物理模拟能力——可还原机器人动力学（如关节运动、重力反馈）、碰撞检测（如机器人与家具的物理交互），而非简单的坐标移动。
- **多机器人支持能力**：提供**RL（强化学习）驱动的控制器API**，直接支持3类主流机器人形态，无需额外开发适配逻辑：
  - 人形机器人：Unitree H1/G1（双足行走，需处理平衡与步态协调）；
  - 四足机器人：Unitree Aliengo（四足步态，易受地面凸起/缝隙影响）；
  - 轮式机器人：Jetbot（差分驱动，运动更稳定但受限于平面地形）。
- **可视化与交互优势**：借助Isaac Sim的渲染能力，用户可自定义视角（如第一人称、第三人称、俯视图）观察机器人运动，便于调试运动异常（如跌倒、卡顿）。

**核心价值**：首次实现“跨机器人形态”的物理真实仿真，避免传统模拟器仅支持“轮式/点代理”的局限，为后续评估“不同机器人的导航性能差异”奠定基础。


### 二、Scenes（场景设计）：解决传统场景的局限性，提升评估泛化性
场景是VLN任务的“环境载体”，VLN-PE针对传统MP3D（Matterport3D）场景的缺陷（视觉多样性不足、物理误差），设计了“修复+新增”的场景体系，具体包括三类场景：

#### 1. 修复后的MP3D场景（对齐传统基准，保证兼容性）
- **原始问题**：MP3D场景由真实室内环境重建而来，存在**地面缝隙**（重建误差导致）——腿部机器人（人形/四足）的腿易卡入缝隙，导致跌倒或卡顿，无法反映真实导航中的“地形适应性”。
- **优化措施**：
  - 将90个MP3D场景转换为USD格式（Isaac Sim支持的通用场景格式）；
  - 人工（含志愿者）修复所有地面缝隙，消除“非导航能力导致的失败”；
  - 对齐VLN-CE（Habitat）基准的坐标系统，确保机器人初始位置、旋转角度与原始标注一致，保证“跨平台对比的公平性”。
- **光照增强**：引入“圆盘照明”（模拟室内面板灯），支持调节光强（如5000lux模拟白天、300lux模拟夜晚），为后续评估“光照对导航的影响”提供条件。

#### 2. 新增场景：弥补MP3D的视觉与物理真实性短板
为评估模型在“非MP3D域”的泛化能力，VLN-PE新增两类场景：
- **GRScenes合成家庭场景**：
  - 来源：10个高质量合成室内场景（来自GRUTopia的GRScenes库）；
  - 优势：视觉保真度高（如纹理细节、家具材质）、物理属性真实（如沙发的柔软度、桌子的硬度，碰撞反馈更贴近现实）；
  - 用途：测试模型从“MP3D场景”迁移到“新风格家庭场景”的适应能力。
- **3DGS重建实验室场景**：
  - 技术：基于3D高斯渲染（3D Gaussian Splatting）技术，对真实实验室环境进行扫描重建；
  - 特点：超写实视觉效果（含人类难以察觉的“渲染噪声”，如墙面细微反光差异）；
  - 用途：评估模型对“真实视觉噪声”的鲁棒性——模拟真实相机拍摄时的图像干扰。

**核心价值**：场景体系覆盖“传统基准对齐（修复MP3D）→ 合成场景泛化（GRScenes）→ 真实噪声鲁棒性（3DGS）”，避免模型仅过拟合MP3D场景，更贴近真实世界的环境多样性。


### 三、Datasets（数据集构建）：适配物理场景，支持多维度评估
数据集是VLN模型训练与评估的“任务载体”，VLN-PE基于经典R2R数据集优化，并新增泛化数据集，确保任务与物理场景匹配：

#### 1. 核心数据集：过滤后的R2R数据集
- **原始基础**：采用VLN领域最经典的R2R数据集（含自然语言指令与对应的导航轨迹）；
- **关键过滤**：移除含“楼梯”的episode——因当前RL控制器暂无法稳定处理“机器人上下楼梯”（避免“控制器短板”影响导航算法评估）；
- **最终划分**（过滤后）：
  - 训练集：8,679个episode（机器人学习“指令-轨迹”映射）；
  - 验证-见过集（Val-Seen）：658个episode（场景与训练集相同，指令不同，评估“同场景泛化”）；
  - 验证-未见过集（Val-Unseen）：1,347个episode（场景与训练集不同，评估“跨场景泛化”）。

#### 2. 新增泛化数据集：评估非MP3D场景的适应能力
为测试模型在“非MP3D场景”的性能，VLN-PE构建两个专属数据集：
- **GRU-VLN10数据集**（针对GRScenes合成场景）：
  - 场景划分：3个训练场景 + 7个测试场景（均为GRScenes中的家庭场景）；
  - episode数量：训练集441个、Val-Seen 111个、Val-Unseen 1,287个；
  - 指令生成：用“模块化LLM（如GPT-4）生成指令 + 人工筛选”，确保指令与场景物体（如“沙发”“餐桌”）匹配，避免歧义。
- **3DGS-Lab-VLN数据集**（针对3DGS实验室场景）：
  - 场景：1个3DGS重建的实验室场景；
  - episode数量：训练集160个、评估集640个；
  - 用途：专门测试模型在“超写实+渲染噪声场景”中的导航能力。

**核心价值**：数据集既保留与传统R2R的兼容性（便于对比历史模型），又新增“合成场景”“真实噪声场景”的评估任务，全面检验模型的泛化边界。


### 四、Metrics（评估指标）：新增物理维度，量化真实导航性能
传统VLN指标仅关注“导航精度与效率”，忽略物理场景中的“机器人稳定性”（如跌倒、卡顿）。VLN-PE设计“5个传统指标+2个物理指标”的完整评估体系，具体如下：

| **指标类型** | **指标名称（缩写）** | **定义与计算方式** | **单位** | **核心作用** |
|----------|------------------|----------------|------|----------|
| 传统指标 | 轨迹长度（TL） | 机器人从起点到停止点的总移动距离 | 米（m） | 衡量路径冗余度（TL越长，路径越低效） |
|          | 导航误差（NE） | 机器人停止位置与真实目标位置的直线距离 | 米（m） | 衡量导航精度（NE越小，定位越准） |
|          | 成功率（SR） | 停止位置在“目标阈值内”（通常设0.5m）的episode比例 | % | 核心导航能力指标（SR越高，任务完成度越好） |
|          | Oracle成功率（OS） | 导航轨迹中“任意点在目标阈值内”的episode比例 | % | 评估“是否接近过目标”（避免因“最后一步偏差”误判能力） |
|          | 路径效率加权成功率（SPL） | 公式：SR × min(1, 真实路径长度/预测路径长度) | % | 平衡“成功率”与“路径效率”（避免“绕远路成功”的低效情况） |
| 物理指标 | 跌倒率（FR） | 机器人因物理失衡跌倒的episode比例（如倾角＞15°（滚转）/35°（俯仰）） | % | 衡量机器人运动稳定性（FR越高，物理适应能力越差） |
|          | 卡顿率（StR） | 机器人50步内“位置变化＜0.2m且航向变化＜15°”的episode比例 | % | 衡量是否陷入“局部死锁”（如卡在家具缝隙、原地旋转） |

**核心价值**：首次将“物理稳定性”纳入VLN评估，避免传统指标“只看结果、不看过程”的缺陷——例如，某模型SR高但FR=50%，在真实部署中仍无法使用；而VLN-PE的指标可全面量化“导航精度+效率+物理安全性”。


### 五、Locomotion Policy（运动策略）：仿真-真实一致，降低部署成本
“运动策略”是连接“导航指令”与“机器人动作”的桥梁，VLN-PE的运动策略设计核心是“仿真与真实机器人的一致性”，具体如下：
- **底层技术**：控制器API基于**最先进的RL运动策略**（如Unitree机器人的专用步态控制算法），针对不同机器人形态优化：
  - 人形/四足机器人：RL策略控制步态、平衡（如避免跌倒）；
  - 轮式机器人：差分驱动策略控制速度与转向。
- **关键特性：仿真-真实迁移一致性**：
  - 仿真中使用的运动模型（如RL训练的步态策略）可**直接部署到真实机器人**（无需重新训练），确保“仿真中表现好的模型，真实环境中性能相近”；
  - 解决传统问题：传统模拟器的运动策略（如“瞬移式移动”）与真实机器人差异大，导致“仿真性能优异、真实部署失效”。
- **开放性**：控制器代码开源，研究者可集成更先进的运动策略（如基于模型预测控制MPC的步态优化），扩展平台能力。

**核心价值**：消除“仿真-真实的运动鸿沟”，降低模型从模拟训练到真实部署的成本，同时为评估“运动策略对导航的影响”（如不同机器人的步态差异是否导致导航性能下降）提供支持。

## 基线
![](/paper/vln-pe-rdp.png)

论文针对“物理真实场景下的视觉-语言导航（VLN）”任务，设计并评估了**三类基准模型**（端到端单步、端到端多步、无训练地图基），均适配“机器人常用的以自我为中心RGB-D针孔相机”硬件约束。

### 一、三类基准模型的核心设计
#### （一）第一类：端到端训练的单步离散动作模型
这类模型是传统VLN的经典方案，核心逻辑是“**逐帧输入→逐步预测离散动作**”（如“前进0.5m”“左转30°”），适配简单机器人控制逻辑，包含3个代表性模型：

##### 1. Seq2Seq（基础基线）
- **核心定位**：VLN领域最基础的端到端框架，无复杂跨模态交互，仅实现“指令+视觉+深度”的简单融合。
- **结构细节**：
  - **输入编码**：
    - 指令编码器：用LSTM将GLoVE词嵌入（指令文本）编码为固定长度特征`I`；
    - 观测编码器：用预训练ResNet50分别编码RGB图像（`V_t`）和深度图（`D_t`），捕捉视觉与空间信息；
  - **动作预测**：用GRU（循环单元）融合`V_t`、`D_t`、`I`及历史状态`h_{t-1}`，输出当前动作`a_t`（公式1-2），动作来自预设离散集合（如5类基础动作）。
- **特点**：结构简单、参数少（36M），但缺乏“指令与视觉的精准对齐”，易因“指令理解偏差”或“视觉特征误判”导致导航错误（如错过目标）。

##### 2. CMA（跨模态注意力增强）
- **核心改进**：在Seq2Seq基础上加入**跨模态注意力机制**，解决“指令与视觉特征脱节”问题。
- **结构细节**：
  - 双GRU设计：
    - 第一个GRU（`h_t^{1st}`）：输入`V_t`、`D_t`和上一步动作`a_{t-1}`，仅跟踪视觉与运动历史；
    - 注意力对齐：用第一个GRU的状态对“指令`I`”“RGB特征`V_t`”“深度特征`D_t`”做缩放点积注意力（公式3-4），得到对齐后的`Î_t`、`V̂_t`、`D̂_t`（确保“指令提到的物体”与“视觉中的物体”匹配）；
  - 第二个GRU（`h_t^{2nd}`）：融合对齐后的跨模态特征+上一步动作+第一个GRU状态，预测最终动作`a_t`（公式5-6）。
- **特点**：相比Seq2Seq，跨模态对齐提升“指令理解精度”，但仍受限于“单步预测”——无法规划后续路径，易陷入“局部最优”（如在目标附近绕圈）。

##### 3. NaVid（视频MLLM驱动）
- **核心突破**：首个基于**视频多模态大语言模型（MLLM）** 的VLN模型，实现“RGB-only导航”（无需深度输入），模拟人类“仅靠视觉判断路径”的行为。
- **结构细节**：
  - 基础框架：基于LLaMa-VID（通用视频MLLM），包含视觉编码器、查询生成器、LLM、跨模态投射器；
  - 输入处理：将“截至当前的t帧图像”编码为视频令牌，用`<HIS>`（历史观测）、`<OBS>`（当前观测）、`<NAV>`（动作输出标识）三类特殊令牌划分输入/输出边界；
  - 动作输出：输出“离散动作类型+定量参数”（如“Forward 1.0m”“TurnLeft 45°”），用正则表达式解析后控制机器人。
- **特点**：
  - 优势：7B大参数带来“世界知识”（如知道“沙发通常在客厅”），物理稳定性好（跌倒率FR低）；
  - 缺陷：RGB-only导致对光照/噪声敏感，且70% episode在目标附近“无意义旋转”（缺乏精准停止机制）。


#### （二）第二类：端到端训练的多步连续预测模型（RDP，论文新提基线）
传统单步模型存在“短视问题”（仅看下一步，忽略长程路径），论文提出**RDP（循环扩散策略）** ，核心逻辑是“**一次预测未来T步连续动作**”（如未来8步的位移`Δx`、`Δy`和偏航角`Δyaw`），更贴近真实机器人的“轨迹规划需求”。

- 解决单步模型的“路径不连贯”；
- 加入历史信息记忆与精准停止机制，适配物理导航。
![](/paper/vln-pe-rdp.png)
- **输入编码**：
  - RGB+指令：用LongCLIP（支持长文本的CLIP变体）分别编码，解决“长指令理解”问题；
  - 深度图：用预训练ResNet50编码，补充空间距离信息；
- **跨模态融合**：通过两个多头多层跨模态注意力模块，对齐“视觉-语言-深度”特征，输出融合特征`c_t`；
- **多步连续动作生成**：
  - 扩散解码器：用Transformer作为扩散策略的解码器，以`c_t`为条件，生成未来T步连续动作（`{Δx_t, Δy_t, Δyaw_t}`）；
  - 去噪过程：遵循DDPM（去噪扩散概率模型），通过迭代去噪（公式7）生成平滑轨迹（避免动作突变导致机器人跌倒）；
- **关键优化**：
  - 历史记忆：用GRU维护历史观测状态，捕捉长程依赖（如“记住5步前见过的目标方向”）；
  - 停止头：新增MLP预测“停止进度”（0→1，0表示刚出发，1表示到达目标），解决“扩散模型无法判断停止时机”的问题。
- **损失函数**：综合“扩散去噪损失”（保证轨迹平滑）和“停止进度损失”（保证精准停止），公式8-9中`λ=10`（重点优化停止精度）。


### （三）第三类：无训练的基于地图的模型（改进VLMaps）
这类模型无需数据训练，核心逻辑是“**LLM解析指令+语义地图定位**”，适合快速部署，论文对经典VLMaps做了关键改进。

#### 1. 原始VLMaps流程
- LLM将自然语言指令解析为“类代码子目标”（如`robot.move in between('sofa', 'chair')`）；
- 用LSeg（语义分割模型）将3D点云体素编码为语义嵌入，通过对比学习对齐“子目标文本嵌入”与“点云语义嵌入”，定位子目标位置；
- LLM生成可执行Python代码，控制机器人移动到子目标。

#### 2. 论文改进：加入VLFM探索策略
- **原始缺陷**：若指令中的地标（如“餐桌”）在初始视角不可见，模型会“卡壳”；
- **改进方案**：当机器人未检测到目标时，执行“边界探索”——转身移动到“未探索区域边界”，用LSeg的图像/文本编码器评估该视角是否包含目标，直到找到地标。
- **特点**：无训练成本，依赖LLM的逻辑推理能力，但受限于“语义地图质量”（若点云重建有误差，定位会偏差）。


### 二、实验结果解读
实验基于**人形机器人（Unitree H1）+RL控制器**，在R2R数据集上评估，核心关注“模拟（VLN-CE）到物理（VLN-PE）的零样本迁移性能”“VLN-PE域内训练提升”“物理指标（FR/StR）差异”三大维度。

#### （一）关键实验设置
- **零样本迁移**：模型在VLN-CE（理想化模拟）训练，直接在VLN-PE（物理真实）测试，评估“模拟-物理鸿沟”；
- **域内训练**：模型在VLN-PE数据上训练，评估“物理环境适配能力”；
- **指标重点**：除传统导航指标（SR成功率、SPL路径效率加权成功率），额外关注物理指标（FR跌倒率、StR卡顿率）。


#### （二）核心发现
##### 1. 零样本迁移：模拟到物理的性能显著下降（图4+表1行2-6）
- **导航指标下降**：
  - CMA-Full（VLN-CE训练）在Val-Unseen集的SR从VLN-CE的~30%降至VLN-PE的16.04%（表1行3），SPL从~20%降至14.63%；
  - NaVid（7B大模型）表现最优，但SR仍下降18%（图4），原因是“模拟中无物理约束（如碰撞、跌倒），物理环境中需兼顾运动稳定性”。
- **物理指标差异**：
  - NaVid的FR（8.61%）远低于Seq2Seq（13.88%）和CMA（15.21%）（表1行2-3、6），因MLLM的世界知识可辅助避障（如“远离桌子边缘避免跌倒”）；
  - CMA的StR（4.31%）高于NaVid（0.45%），因CMA的单步预测易导致“卡在家具缝隙”。

##### 2. 域内训练：VLN-PE数据微调大幅提升性能（表1行7-11）
- **导航指标提升**：
  - CMA+（VLN-PE微调）在Val-Seen集的SR达28.72%，SPL达24.24%，远超零样本CMA（SR=11.32%，SPL=10.20%）（表1行4、11）；
  - RDP（多步连续预测）从零训练时，Val-Unseen集SR=21.98%，优于Seq2Seq（15.89%）和CMA（18.78%）（表1行7-9），验证“多步预测比单步更适应物理导航”。
- **物理指标优化**：
  - CMA+的FR=20.21%，低于零样本CMA（21.12%），因微调适配了机器人运动动力学（如“调整步长避免跌倒”）。

##### 3. 相机高度敏感：机器人形态影响导航性能（表1行4-5）
- 对比CMA（相机高度默认值）与CMA-Height1.8（相机高度1.8m，匹配人形机器人身高）：
  - Val-Unseen集SR从10.31%提升至13.58%，FR从15.21%降至16.11%（轻微上升，因高度调整需适应）；
  - 核心原因：相机高度决定“视觉视野”——1.8m高度更易看到“沙发、桌子”等地标，减少目标误判。

##### 4. 无训练模型：VLMaps*的局限性（表1行12）
- VLMaps*（改进后）在Val-Unseen集SR=20.00%，SPL=12.70%，低于域内训练的CMA+（SR=23.31%）；
- 缺陷：依赖语义地图精度，若3D点云有噪声（如3DGS场景），定位误差会增大，且探索策略耗时较长（平均轨迹长度TL=15.73m，远超其他模型）。


#### （三）模型性能排名（Val-Unseen集，综合SR+FR+SPL）
1. **域内训练最优**：CMA+（SR=23.31%，FR=18.63%，SPL=19.66%）——跨模态注意力+物理环境微调，平衡导航精度与稳定性；
2. **多步预测潜力**：RDP（SR=21.98%，FR=18.75%，SPL=16.44%）——连续轨迹更平滑，适合复杂物理场景；
3. **零样本最优**：NaVid（SR=22.42%，FR=8.61%，SPL=18.58%）——大模型世界知识提升稳定性，但缺乏微调导致停止精度不足；
4. **无训练备选**：VLMaps*（SR=20.00%，FR=23.00%，SPL=12.70%）——无需训练但效率低，适合快速验证场景。


### 三、核心结论
1. **模拟-物理鸿沟不可忽视**：零样本迁移时所有模型性能下降，需通过“域内微调”或“物理约束融入训练”缩小差距；
2. **模型设计需适配物理特性**：
   - 跨模态融合（如CMA）提升指令-视觉对齐精度；
   - 多步连续预测（如RDP）减少动作突变，降低跌倒率；
   - 大模型世界知识（如NaVid）辅助避障，优化物理稳定性；
3. **硬件参数影响显著**：相机高度、控制器类型（RL/传统）需与模型训练条件一致，否则会导致性能骤降（如控制器不一致时SR下降8%+）。

## 实验
### 一、问题1：VLN-CE模型向VLN-PE迁移的效果如何？
VLN-CE（如Habitat平台）是理想化模拟环境，VLN-PE是物理真实环境，实验通过“零样本迁移”“域内训练微调”“MP3D域外泛化”三类场景，评估两者的**模拟-物理鸿沟**。

#### 1. 实验设计
- **模型**：Seq2Seq、CMA、NaVid（VLN-CE训练的经典模型）、VLMaps（无训练地图模型）；
- **平台**：人形机器人Unitree H1（RL控制器），数据集为R2R（基础）、GRU-VLN10（MP3D域外合成场景）、3DGS-Lab-VLN（超写实渲染场景）；
- **关键对比**：零样本迁移（VLN-CE训练→VLN-PE测试） vs 域内训练（VLN-PE数据从头训） vs 域内微调（VLN-CE模型用VLN-PE数据微调）。


#### 2. 核心结果（表1、图4、表2、表3）
##### （1）零样本迁移：性能显著下降，大模型泛化略优
- **导航指标骤降**：直接迁移时，所有模型SR（成功率）均下滑：
  - Seq2Seq-Full（#2）：Val-Unseen集SR从VLN-CE的~30%降至15.00%（降10%）；
  - CMA-Full（#3）：SR从~32%降至16.04%（降16%）；
  - NaVid（#6，7B大模型）：SR从~40%降至22.42%（降18%），但仍是零样本最优（图4）。
- **原因**：VLN-CE无物理约束（如无碰撞、无跌倒），模型未学习“运动稳定性”；而VLN-PE中机器人需应对真实动力学（如平衡、避障）。

##### （2）域内训练/微调：大幅弥补迁移差距
- **域内从头训优于VLN-CE迁移**：CMA（#8，VLN-PE从头训）Val-Seen集SR=21.12%，远超零样本CMA-Full（#3，15.50%）；
- **微调效果更显著**：CMA+（#11，VLN-CE模型用VLN-PE数据微调）Val-Seen集SR=28.72%、SPL=24.24%，不仅超过零样本NaVid（SR=21.58%），还解决了“过拟合VLN-CE”问题。
- **相机高度适配提升迁移性**：CMA-Height1.8（#5，相机高度1.8m匹配人形机器人）Val-Unseen集SR=13.58%，优于默认高度1.2m的CMA（#4，10.31%），证明“硬件参数（相机高度）需与训练条件对齐”。

##### （3）MP3D域外泛化：小模型微调优于大模型零样本
- **GRU-VLN10场景（表2）**：仅用3个场景441个episode微调的RDP（#5）Val-Unseen集SR=28.52%，远超NaVid零样本（#1，18.64%），平均SR提升18.86%；
- **3DGS-Lab-VLN场景（表3）**：NaVid完全失效（SR=5.81%，持续旋转），因3DGS渲染噪声（人类不可见但干扰RGB感知）；而微调后的CMA-CLIP（#3，SR=24.88%）、RDP（#5，SR=30.63%）鲁棒性更强。

##### （4）避障优势与缺陷：NaVid的“稳定性-精度”矛盾
- **优势**：NaVid的FR（跌倒率=8.61%）、StR（卡顿率=0.45%）远低于CMA（FR=15.21%、StR=3.41%），因MLLM的“世界知识”（如“远离桌子边缘避障”）；
- **缺陷**：70% episode在目标附近“无意义旋转25步以上”，说明大模型缺乏“精准停止机制”。


#### 3. 结论
- VLN-CE模型直接迁移到物理环境会因“过拟合模拟”导致性能骤降；
- **域内微调**（用VLN-PE数据）是缩小模拟-物理鸿沟的高效手段；
- 大模型（如NaVid）虽有避障优势，但对“渲染噪声”敏感，且停止精度不足；小模型微调后在域外场景更鲁棒。


### 二、问题2：物理控制器如何影响性能？
物理控制器是“导航指令→机器人动作”的桥梁，实验聚焦“数据收集与评估时控制器是否一致”，验证其对腿式机器人（人形）性能的影响。

#### 1. 实验设计
- **模型**：CMA（从头训练）；
- **变量**：数据收集时是否启用物理控制器（Collect w/ loco：✓/✗）、评估时是否启用物理控制器（Eval w/ loco：✓/✗）；
- **核心指标**：FR（跌倒率）、StR（卡顿率）、SR（成功率），重点关注“控制器一致性”的影响。


#### 2. 核心结果（表4）
| 实验组 | Collect w/ loco | Eval w/ loco | Val Seen SR | Val Unseen SR | Val Seen FR | Val Unseen FR |
|--------|-----------------|--------------|-------------|---------------|-------------|---------------|
| 1      | ✗               | ✗            | 20.21%      | 21.31%        | 0.00%       | 0.00%         |
| 2      | ✓               | ✗            | 18.54%      | 15.90%        | 0.00%       | 0.00%         |
| 3      | ✗               | ✓            | 12.92%      | 13.51%        | 31.76%      | 25.69%        |
| 4（最优） | ✓             | ✓            | 21.12%      | 18.78%        | 23.40%      | 18.63%        |

- **控制器一致时性能最优**：组4（收集/评估均用控制器）Val Seen SR=21.12%、FR=23.40%，是所有组中SR最高、FR最低的；
- **控制器不一致时性能骤降**：组3（收集无控制器/评估有控制器）SR仅12.92%（比组4低8.2%），FR飙升至31.76%（比组4高8.36%），因训练时未学习“物理动力学反馈”（如行走颠簸），评估时机器人无法适应；
- **控制器数据提升稳定性**：组4的FR（23.40%）低于组3（31.76%），说明“用控制器收集的数据”能让模型学习避障、平衡，减少跌倒。


#### 3. 结论
- **控制器一致性是关键**：训练（数据收集）与评估时使用相同物理控制器，是保证物理环境性能的核心；
- 物理控制器数据能“注入真实动力学信息”，显著提升腿式机器人的运动稳定性（降低FR/StR），对物理导航至关重要。


### 三、问题3：跨具身数据如何影响适应性？
“具身”指机器人形态（人形/四足/轮式），实验验证“单一机器人数据vs跨机器人数据”对模型泛化能力的影响，目标是实现“One-for-All”（一个模型适配所有机器人）。

#### 1. 实验设计
- **模型**：CMA；
- **机器人类型**：人形（相机高度1.8m）、四足（0.5m）、轮式（0.3m）、Flash（模拟Habitat平滑运动）；
- **数据组合**：VLN-CE数据、单一机器人数据、跨机器人数据（人形+四足+轮式）；
- **关键指标**：Val Unseen SR（跨场景泛化）、OS（Oracle成功率）。


#### 2. 核心结果（表5、图5）
##### （1）模型对“运动动态+相机高度”高度敏感
- 四足机器人零样本迁移时性能最差：VLN-CE数据训练的CMA（仅✓VLN-CE），Val Unseen SR仅4.73%，因相机高度0.5m（远低于人形1.8m），视觉视野受限（无法看到高处地标）；
- 人形Flash模式最优：Flash模拟Habitat平滑运动，零样本SR=21.74%，接近VLN-CE性能，证明“运动模式越接近训练环境，迁移性越好”。

##### （2）单一机器人数据微调显著提升适配性
- 人形机器人：用自身数据微调后（✓VLN-CE+✓人形），Val Unseen SR从16.04%升至18.78%；
- 四足机器人：微调后SR从4.73%升至17.00%（提升12.27%），说明“特定机器人的视角数据”能解决“相机高度/运动模式不匹配”问题。

##### （3）跨具身联合训练实现“One-for-All”
- 跨数据训练（VLN-CE+人形+四足+轮式）的模型，在所有机器人上均达最优：
  - 人形Val Unseen SR=26.87%（比单一数据高8.09%）；
  - 四足SR=23.83%（比单一数据高6.83%）；
  - 轮式SR=20.02%（比单一数据高7.99%）；
- **原因**：跨具身数据提供“多视角信息”（如人形的高处视野、四足的地面视野），模型能更全面理解环境，同时数据量增加减少过拟合。


#### 3. 结论
- VLN模型对机器人的“相机高度”“运动动态”极敏感，单一形态数据训练的模型泛化性差；
- **跨具身联合训练**是实现“One-for-All”模型的关键，既能利用多视图学习提升环境理解，又能适配不同机器人的物理特性。


### 四、问题4：观测条件（光照）如何影响准确性？
观测条件的核心是“光照变化”，实验模拟“白天/夜晚/机器人自发光”三种场景，验证模型对光照干扰的鲁棒性。

#### 1. 实验设计
- **光照设置**（图6）：
  - DL5000：圆盘灯（模拟白天，光强5000lux）；
  - DL300：圆盘灯（模拟夜晚，光强300lux）；
  - CL：相机灯（机器人自带灯，易导致墙面反光、侧面昏暗）；
- **模型对比**：NaVid（RGB-only，仅用图像）、CMA/RDP（RGB+深度，多模态）；
- **关键指标**：Val Unseen SR（光照变化下的导航精度）。


#### 2. 核心结果（表6）
| 模型   | 光照   | Val Unseen SR | SR下降幅度（vs DL5000） |
|--------|--------|---------------|--------------------------|
| NaVid  | DL5000 | 22.42%        | -                        |
|        | DL300  | 9.95%         | 12.47%（降幅最大）       |
|        | CL     | 11.17%        | 11.25%                   |
| CMA    | DL5000 | 18.78%        | -                        |
|        | DL300  | 17.37%        | 1.41%（降幅最小）        |
|        | CL     | 17.04%        | 1.74%                    |
| RDP    | DL5000 | 21.98%        | -                        |
|        | DL300  | 22.27%        | 0.00%（基本无降）        |
|        | CL     | 22.78%        | 0.00%（反而微升）        |

- **RGB-only模型（NaVid）对光照极敏感**：夜晚（DL300）时SR从22.42%降至9.95%（降12.47%），因光照不足导致图像纹理模糊，模型无法识别地标；
- **RGB+深度模型（CMA/RDP）鲁棒性强**：CMA在DL300下SR仅降1.41%，RDP甚至无下降，因深度图不受光照影响（能稳定提供空间距离信息），弥补了RGB图像的缺陷。


#### 3. 结论
- 光照变化是物理环境中常见的观测干扰，**RGB-only模型**（如NaVid）因依赖视觉纹理，鲁棒性差；
- **多模态融合（RGB+深度）** 是提升光照鲁棒性的关键，深度信息能提供“不受光照影响的空间结构”，确保导航精度稳定。
